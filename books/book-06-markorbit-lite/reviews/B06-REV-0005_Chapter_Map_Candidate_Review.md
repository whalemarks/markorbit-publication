## 7. Cross-book duplication review

### Book 04

Book 04 already defines general Workplace, Product embedding, AI placement, data boundaries, Artifact and network architecture.

B06-TOC-V0.1 references those boundaries but focuses on Lite-specific behavior:

- daily cockpit;
- value candidates;
- customer-development journeys;
- professional work-product production;
- cases and memory;
- Lite-specific Handoffs and MVP.

Result: no general Workplace rewrite required.

### Book 05

Book 05 owns the full MarkReg trademark lifecycle.

Book 06 limits MarkReg coverage to:

- qualified need;
- Launch or Continuation Handoff;
- destination revalidation;
- Return and next value.

Result: no MarkReg lifecycle duplication.

### Book 07 / MGSN

Book 06 covers Capability Need and returned candidate presentation only. Trust, Routing, provider appointment and collaboration depth remain outside Book 06.

Result: no second MGSN.

## 9. Writing-order review

- each wave has a coherent thesis;

- the user does not need one branch per chapter;
