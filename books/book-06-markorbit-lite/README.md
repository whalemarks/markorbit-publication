## Product definition

> MarkOrbit Lite turns fragmented customers, trademarks, cases, Knowledge and market signals into timely customer actions, trustworthy professional work products, governed MarkReg/service Handoffs and reusable business capability.

## Four core loops

```text
Customer and Service Growth
Professional Work Products
Professional Memory and Business Assets
MarkOrbit Ecosystem Handoff
```

## Today

## Primary journey

```text
authorized customer/trademark portfolio
→ explainable Service-Value Candidate
→ customer work product and follow-up
→ final user confirmation
→ response and qualification
→ MarkReg / Opportunity / service Handoff
→ returned result
→ next value and reusable capability
```

## MVP 0

```text
Customer Opportunity-to-Governed-Service Loop
```

Content generation and Prospect Candidate supply may support the experiment, but content count or lead count alone cannot pass.

## Work-product chain

```text
authorized data / Knowledge / case context
→ Structured Content
→ Artifact Draft and Version
→ Review
→ Render and optional Edit
→ Delivery / Communication / Publish preparation
→ user-confirmed or governed operation
→ typed result
```

```text
Content ≠ Artifact
Artifact ≠ Document or Evidence
Render complete ≠ approved
Publish Package ready ≠ published
```

## Ecosystem boundary

Lite prepares typed Handoffs to:

- MarkReg;
- MGSN;
- Human Review;
- Communication Service;
- Opportunity Service;
- Task/Execution;
- Delivery/Publish;
- Knowledge Governance.

Destinations revalidate and return sourced results. Lite does not absorb their lifecycle ownership.

## Historical V1 input

The early V1 model contributes requirements around unified objects, private data, platform/private separation, audit and Local-ready design.

It is not the current schema. `B06-SPEC-0004` classifies its concepts as KEEP, REFRAME, PRODUCT-LOCAL, MOVE TO OWNING SERVICE, DEFER or REJECT.
