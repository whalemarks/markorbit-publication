## Purpose

Earlier provider, pricing, assignment and supply-center concepts contain useful demand evidence, but they must be reconciled with the current platform-owned hub model.

## Design Consequence

The historical material supports a strong platform model when reframed around:

```text
independent private spaces
→ controlled projection into MGSN
→ platform supply
→ platform procurement
→ platform routing
→ platform funds control
→ platform fulfillment governance
→ Return to private space
```

It does not support an unrestricted public marketplace, a user-owned provider network or participant-to-participant connectivity.

## Implementation Boundary

No historical table or field name is accepted as current schema authority.

The Book 07 Charter and future controlled baseline must first determine:

- authoritative records;
- relationship to Book 02 Core;
- formal state ownership;
- private-space projection and Return contracts;
- interface and Handoff boundaries;
- legal funds model;
- Product versus network responsibility.
