# B07-PLN-0012 — Workplace Sovereignty Upstream Reconciliation Notice

## Status


## Purpose

This record prevents Book 07 from advancing into a Controlled Product Baseline while the Workplace sovereignty and MGSN boundary are under Owner review through `MO-ADR-WSP-001`.

It does not delete or reject the existing Charter Candidate. It reclassifies that Charter as a reconciliation input pending the upstream architecture decision.

## Confirmed Upstream Issue

The current architecture baseline does not yet fully distinguish:

- Workplace business sovereignty;
- Core semantic authority;
- Owning Service formal state authority;
- Product Installation;
- Product Projection;
- Workplace MGSN connection;
- platform-owned MGSN network.

Book 07 depends on those distinctions.

## Reconciliation Direction

After Owner decision on `MO-ADR-WSP-001`, the Book 07 Charter must explicitly use:

```text
Originating Workplace
→ authorized MGSN demand projection
→ platform-owned MGSN network
→ controlled provider instruction
→ Execution Provider Workplace
→ governed Return
→ Originating Workplace
```

It must preserve:

- the originating Workplace's Customer, Order, Matter and commercial responsibility;
- the platform's ownership or governance of MGSN provider supply, procurement, routing, funds, fulfillment and platform Trust;
- the provider Workplace's professional responsibility and provider-side records;
- Owning Service authority over formal state;
- the distinction between MGSN connection/projection and MGSN network resources.

## Existing Charter Treatment

- platform-owned managed network;

- R1–R3 route model;
- multi-provider portfolios;
- default recommendation and bounded choice;
- contextual demand/supply roles;
- procurement and pricing direction;
- funds and fulfillment governance;
- provider admission, Trust and replacement;
- operator-assisted MVP.

The following require reconciliation:

- generic demand-side/provider-side language;
- Provider Profile ownership;
- Organization/Workplace role in MGSN;
- cross-Workplace Handoff and Return;
- collaboration-state ownership;
- private relationship history versus platform provider resources;
- Product Installation and gateway boundaries.
