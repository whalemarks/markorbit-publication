# Book 03 Governance

Book 03 consumes Book 02 Core contracts. It may describe how those contracts are coordinated during operational execution, but it must not rewrite Book 02 or define new Core architecture.

Book 03 defines execution coordination, not Core architecture. It covers governed workflows, tasks, reviews, communications, event traces, and human-AI collaboration boundaries as execution-layer concerns.

Book 03 does not define product UI. Product surfaces, including Workplace, belong to future product-system work and are not the main subject of this book.

Codex may scaffold, organize, index, and prepare review materials for Book 03. Codex must not define architecture, expand MVP scope, approve professional decisions, send communications, submit filings, mutate protected state, or define professional truth.
