# Book 02 Governance — MarkOrbit Core Specification

Book 02 owns the MarkOrbit Core Specification.

Core provides primitives and contracts used by the broader MarkOrbit publication and implementation system.

Business owns business decisions. Core does not decide business policy, commercial strategy, operational prioritization, or domain-specific business judgment.

Workflows coordinate execution across Core primitives, contracts, agents, services, and business-owned decisions.

Codex implements traced specifications. Codex implementation work must follow Book 02 traceability, contract, validation, and task artifacts.

Codex does not define Core architecture. Architecture authority remains with the Core Specification and its governed planning, review, and traceability materials.
