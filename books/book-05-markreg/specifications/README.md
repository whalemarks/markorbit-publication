# Book 05 Controlled Specifications

These records provide shared Product contracts used across multiple chapters. They prevent later chapters from repeating constitutional explanations and allow the manuscript to reference one controlled definition.

## Current Specifications

- `B05-SPEC-0001_Product_Artifact_and_Decision_Map.md` — canonical artifacts, decisions, lineage, versioning, supersession, change propagation, and service-family conformance.
- `B05-SPEC-0002_Reference_Journeys.md` — one end-to-end multi-jurisdiction filing journey and one contrasting renewal/ownership journey.
- `B05-SPEC-0003_Conformance_Scenarios_and_User_Surfaces.md` — observable Given/When/Then cases and the CH08–CH22 user-surface matrix.
- `B05-SPEC-0004_Jurisdiction_Pack_and_Commercial_Control_Contract.md` — minimum jurisdiction intelligence, rule evaluation, fee, currency, tax, discount, payment, and refund controls.

## Reference Rule

Part I retains the full constitutional explanation. Parts II–VII should reference named constitutional rules and these specifications, then add chapter-specific Product meaning rather than restating the architecture boundary.
