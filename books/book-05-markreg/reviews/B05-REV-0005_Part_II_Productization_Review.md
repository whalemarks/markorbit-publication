## 5. Product Progression

The revised Part II now progresses as:

```text
Business objective and authorized context
→ Need Brief
→ jurisdiction and route Recommendation Set
→ country-bundle Option Set
→ filing-unit candidates
→ applicant and authority context
→ class candidates
→ goods/services versions
→ search and risk decisions
→ Product inputs for Proposal and Quote
```

Each chapter adds a new Product decision or artifact. Later chapters reference the shared constitutional rules rather than restating the entire architecture boundary.

## 6. User Experience Result

Each chapter now states:

- the user question being resolved;
- minimum new information requested;
- context reused;
- Product output;

- professional intervention point;
- primary user action;
