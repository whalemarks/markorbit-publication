## Core Question

> How does an independent professional organization establish its own operating orbit and consume MarkOrbit Core, Execution, knowledge, intelligence, capabilities, products and network services through its Workplace?

Chinese working expression for explanatory context only:

> 一个独立专业机构如何建立自己的运行轨道，并通过Workplace消费MarkOrbit的Core、Execution、Knowledge、Intelligence、Capability、Products和Network？

## Scope Summary

Book 04 may define Workplace as an independent organizational Orbit, organization identity and context, people and accountability, private knowledge and authorized AI context, operating surfaces, data boundaries, consumption of Information, Knowledge, Intelligence, Capability and Skill, AI placement, recommendations, handoff to governed Execution, Product Architecture principles, product embedding and cross-product handoffs, architectural profiles for Lite, MarkReg and MGSN, Artifact and Delivery boundaries, network participation, routing, capability evidence, trust and ecosystem learning without centralization.

## Non-Goal Summary

Book 04 is not a replacement for Book 01, a semantic amendment to Book 02, a replacement for Book 03, a product feature catalog, a Lite PRD, a MarkReg PRD, an MGSN marketplace specification, an Agent framework tutorial, a Prompt engineering manual, an Information & Value Stack manual, a microservice decomposition plan, a database schema specification, an API implementation specification, a deployment architecture, a code repository design or a production-readiness authorization.

Book 04 must not redefine Core Objects or Services, redefine Workflow, Task, Human Review or Event ownership, authorize autonomous AI professional action, authorize external Communication send, authorize filing, submission, payment, provider instruction or official recordal, treat Distillery candidates as approved knowledge or capability, treat recommendations as formal Tasks or Opportunities, treat Artifact as identical to Document, treat MGSN as an open bidding marketplace, or treat Workplace as a centrally owned platform account.
