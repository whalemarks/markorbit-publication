# Book 07 v2 — Repetition and Density Reduction Plan

## 1. Objective

The first full draft intentionally reinforced constitutional boundaries while it was written in batches. The publication pass must now preserve those boundaries without making the reader feel that the same warnings are being re-taught in every chapter.

The goal is not aggressive shortening. The goal is better allocation of explanation.

```text
Primary chapter
→ full explanation

Later chapter
→ concise reminder or cross-reference
```

## 3. Repeated lock statements

Retain these as book-level anchors:

```text
Recommendation ≠ Appointment
Provider Acceptance ≠ Customer Instruction
Payment ≠ Performance
Provider Return ≠ Official Truth
Return Accepted ≠ Formal State Updated
Data Portability ≠ Authority Portability
```

Other inequality blocks should be reduced where prose already makes the distinction clear.

Target:

- no more than one major inequality block in most chapters;
- additional blocks only where the chapter introduces a genuinely new boundary;
- avoid repeating the same three-line formula in adjacent chapters.

## 4. Long-list treatment

Move reference-heavy inventories into tables or appendices where possible.

Highest-priority candidates:

- admission states;
- Service Package fields;
- authority layers;
- funds checkpoints;
- Return types;
- Incident severity and findings;
- dispute forums;
- metric families.

Keep lists in the narrative only when sequence or completeness is essential to the argument.

## 5. Failure-mode treatment

Failure modes are one of the manuscript’s strengths, but many chapters contain five or more examples after already detailed explanations.

Editorial rule:

- retain two or three representative failure modes in each chapter;
- merge near-duplicates;
- move comprehensive failure inventories to a reference appendix;
- preserve the recovery action for every retained failure mode.

## 6. Chapter transition treatment

Each chapter should end by creating the next question.

The transition should normally be one short paragraph, not another full summary of all locks.

Eight Part transitions should carry the larger shifts:

```text
Problem → Constitution
Constitution → Supply
Supply → Demand
Route → Engagement
Engagement → Commercial Governance
Commercial Baseline → Fulfillment
Fulfillment → Trust and Failure
Network Learning → Infrastructure
```

## 7. Density targets

This pass should aim for:

- shorter opening sections;
- fewer consecutive abstract sections;
- more operating examples before taxonomies;
- tables for comparisons;
- one principal model per chapter;
- fewer repeated disclaimers;
- clearer separation between argument and reference material.

No numeric word-reduction target should be imposed before chapter-level review. Some chapters may become shorter; others may gain a short example or bridge while losing reference density.

## 8. Protected material

Do not remove distinctions that protect:

- customer relationship ownership;
- professional qualification and authority;
- Provider Acceptance;
- protected-action approval;
- funds and official-fee separation;
- Evidence and formal-state separation;
- correction history;
- replacement authority;
- due process;
- portability boundaries;
- AI authority limits.

Reduction must improve readability without weakening governance.

## 9. Execution order

Recommended editorial sequence:

1. Add Introduction and eight Part introductions.
2. Normalize titles, capitalization and defined terms.
3. Edit CH01–CH08 for opening rhythm and supply-side repetition.
4. Edit CH09–CH16 for route and engagement continuity.
5. Edit CH17–CH24 for cost, funds, Return and recovery repetition.
6. Edit CH25–CH32 for Trust, governance and synthesis density.
7. Create appendices from removed reference inventories.
8. Perform final cross-book and authority review.

## 10. Completion criteria

The density pass is complete when:

- every concept has one clear primary home;
- adjacent chapters do not re-teach the same model;
- chapter openings are reader-facing;
- chapter endings advance the argument;
- long inventories are presented as reference material;
- failure modes remain concrete and recoverable;
- authority and ownership locks remain intact;
- CH32 is the only complete synthesis of the whole system.
