## 4. Chapter results

### CH01

Establishes global service capability as an end-to-end operating problem rather than provider discovery.

Key lock:

```text
Global supply visibility
≠ global operating capability
```

The chapter follows Need understanding, authorized Projection, eligible supply, route choice, Provider Acceptance, fulfillment, evidence, reconciliation and formal-state update.

### CH02

Explains why private spreadsheets, email memory and personal networks remain useful but become structurally fragile at scale.

Key lock:

```text
Relationship history
≠ current eligibility
```

The chapter covers stale prices, changing contacts, organizational continuity, key-person risk, Trust compression, circumvention, recovery and the strengths that MGSN should preserve from existing relationships.

### CH03

Defines the Product distinction between a managed network and an open marketplace.

Key lock:

```text
Marketplace liquidity
≠ managed network quality
```

The chapter rejects maximum visible choice, lowest-price routing, public-star shortcuts, automatic appointment and opaque platform preference as constitutional defaults.

### CH04

States the Product Constitution and separates:

- Originating Workplace;
- MGSN Connection;
- platform-owned MGSN Network;
- Execution Provider Workplace;
- Core and Owning Services;
- Book 03 Execution;
- professional and official authority.

Key lock:

```text
Managed coordination
≠ unlimited platform authority
```
