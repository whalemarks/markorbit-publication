## 3. Editorial purpose

Part III originally contained the correct demand-and-routing architecture but repeated the same fields, outcome lists, AI restrictions and failure catalogs across four chapters.

The revision gives each chapter one primary question:

```text
CH09
What may leave the Originating Workplace?

CH10
Which qualified routes are usable now?

CH11
Which usable routes should be presented as meaningful alternatives?

CH12
Why is one route recommended and how may the user challenge it?
```

The resulting reader chain is:

```text
Private Workplace Need
→ Purpose-limited Projection
→ Current Eligibility
→ Bounded Candidate Route Set
→ Explainable Recommendation
→ User Disposition
```
