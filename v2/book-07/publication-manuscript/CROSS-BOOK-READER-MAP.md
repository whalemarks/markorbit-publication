# Cross-book Reader Map

Book 07 depends on concepts developed elsewhere in the MarkOrbit series. This map helps readers understand those dependencies without turning Book 07 into a summary of the other six books.

## Book 01 — MarkOrbit as a Professional Operating System

Book 01 explains why professional work requires a system that separates facts, intelligence, capability, authority, execution and evidence.

Book 07 relies on Book 01 for:

- the distinction between capability and a feature;
- governed learning without silent canonization;
- the separation between the Professional Contribution Network and MGSN;
- the principle that global fulfillment requires accountable routes, not directories.

Book 07 adds:

- the operating model for external Provider supply;
- service-specific route formation;
- Provider Acceptance, Return, correction and replacement;
- network Trust and recovery.

Book 01 remains the strategic and constitutional overview. Book 07 is the detailed MGSN operating book.

---

## Book 02 — Core and Shared Meaning

Book 02 governs the shared semantic grammar used across MarkOrbit Products.

Book 07 relies on Book 02 for:

- Organization, Workplace and identity distinctions;
- Evidence, Claim, Decision and Formal State separation;
- Projection, Handoff, Return and Correction boundaries;
- public, shared and Product-local record classes;
- versioning and controlled semantic change.

Book 07 does not own:

- Core semantic admission;
- controlled values;
- public contract versioning;
- shared semantic migration.

A term appearing in Book 07 does not become a new Core Object merely because it is useful to the Product.

---

## Book 03 — Governed Execution

Book 03 explains how a governed Capability Request becomes bounded work through Eligibility, Assignment, review, approval, Apply, Return and reconciliation.

Book 07 relies on Book 03 for:

- execution risk and authority boundaries;
- Human–AI collaboration limits;
- External Protected Action controls;
- unknown-state handling and safe retry;
- Evidence-bearing handoff, Return and recovery;
- the rule that execution learning cannot rewrite truth.

Book 07 adds the MGSN-specific route around that execution model:

```text
Need Projection
→ Candidate Route
→ Recommendation
→ Provider Allocation
→ Provider Acceptance
→ Instruction Package
→ Fulfillment
→ Provider Return
→ Correction or Replacement
```

Book 03 owns governed runtime principles. Book 07 owns the MGSN network route and its Product-local controls.

---

## Book 04 — Workplace Sovereignty

Book 04 defines Workplace as the sovereign business context in which Products operate.

Book 07 relies on Book 04 for:

- customer relationship ownership;
- minimum necessary Projection;
- Product Installation and Product-local state;
- Assignment-scoped access;
- private memory and learning boundaries;
- portability and exit;
- the distinction between the MGSN Connection and the platform-owned MGSN Network.

The Originating Workplace keeps:

- its customer relationship;
- internal Matter context;
- private Knowledge;
- pricing and commercial strategy;
- the decision to use, reject or leave an MGSN route.

MGSN governs the route it represents. It does not absorb the Workplace.

---

## Book 05 — MarkReg Service Delivery

Book 05 shows how a complete trademark-service Product governs intake, search, filing, prosecution, registration, maintenance, transactions, evidence, recovery and commercial outcomes.

Book 07 relies on Book 05 for practical examples of:

- formal-service readiness;
- Provider appointment and filing authority;
- official-fee separation;
- typed Provider Returns;
- correction and formal-state validation;
- service Outcome evidence;
- white-label and co-delivery boundaries.

Book 07 does not own MarkReg formal state. A Provider Return routed through MGSN must still be validated by the applicable Owning Service before it changes a Matter’s formal business state.

```text
MGSN Return accepted
≠ MarkReg formal state updated
```

---

## Book 06 — Lite, Daily Work and Capability Growth

Book 06 defines Lite as the daily growth, trademark supply, buyer demand, opportunity and Capability Workplace for professionals and small teams.

Book 07 relies on Book 06 for:

- user-facing guidance and demand clarification;
- Capability Profile and Agent Capability Twin boundaries;
- learning-in-work and AI Mentor limits;
- trademark supply and opportunity Handoffs;
- the separation between Recommendation and commitment;
- the distinction between user Capability evidence and Provider performance evidence.

Lite may help a user understand a Need or prepare a Handoff. MGSN governs eligible external supply and fulfillment. MarkReg governs the full service lifecycle when a formal trademark service is opened.

---

## Book 07 — MGSN

Book 07 owns the detailed operating argument for the Mark Global Service Network.

Its primary responsibilities are:

- Provider Organization admission;
- Provider Supply Capability representation;
- Service Package admission;
- current Eligibility;
- Candidate Route formation;
- explainable Recommendation;
- Provider Allocation and Acceptance;
- versioned Instruction Packages;
- observable fulfillment;
- Provider Return requirements;
- correction, replacement and recovery;
- service-specific MGSN Trust;
- Incident and Dispute routing;
- network learning without public ranking;
- portability without lock-in.

Book 07 does not own:

- shared Core semantics;
- Workplace customer relationships;
- statutory Professional Qualification;
- Matter-specific customer authority;
- universal funds custody;
- official records;
- formal state held by an Owning Service;
- shared Capability version decisions.

## Responsibility summary

| Layer | Primary responsibility |
|---|---|
| Core | Shared semantics and controlled contracts |
| Capability Governance | Shared Capability evaluation and version change |
| Workplace | Customer, business context, private Knowledge and local decisions |
| Execution | Governed runtime coordination |
| Lite | Daily work, guidance, opportunity and user Capability growth |
| MarkReg | Complete trademark-service lifecycle and formal business state |
| MGSN | External Provider supply, route formation, fulfillment and recovery |
| Official Authority | External official state |

## Reader shortcut

For the central Book 07 chain, use the following reading path:

```text
Book 04 — Who owns the business context?
→ Book 02 — What shared terms mean
→ Book 03 — How governed work executes
→ Book 07 — How external Provider routes form and recover
→ Book 05 — How a formal trademark service validates its state
→ Book 06 — How users discover Needs and grow Capability in daily work
```

The books cooperate through typed boundaries. None of them becomes the universal owner of customers, authority, truth or professional judgment.
