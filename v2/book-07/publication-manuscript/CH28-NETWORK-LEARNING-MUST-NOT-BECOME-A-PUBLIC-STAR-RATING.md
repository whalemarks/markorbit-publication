# CH28 — Network Learning Must Not Become a Public Star Rating

A managed network should improve with every completed, corrected, disputed and recovered engagement.

But learning does not require public star ratings, and it does not authorize MGSN to rewrite shared Capability canon.

```text
Network Learning
≠ Public Popularity Ranking
≠ Universal Provider Score
≠ Automatic Capability Upgrade
```

## 1. What the network should learn

Learning can occur at four levels.

### Supply learning

Where Provider Supply Capability gaps, concentration, stale Evidence, ambiguous Packages and urgent-capacity shortages exist.

### Routing learning

Which Need fields change Eligibility, why users rematch and where Unknowns should surface earlier.

### Fulfillment learning

Which Milestones predict delay, which Evidence is frequently missing, where Returns require clarification and where replacement fails.

### Governance learning

Which Incidents reveal control-design failure, which Disputes arise from ambiguous terms and which restrictions are too broad or too weak.

The goal is to improve the system, not merely judge individuals.

## 2. Learning units must remain bounded

Learning should attach to the smallest useful context:

- Provider Organization or office;
- responsible professional;
- Provider Supply Capability;
- Service Package;
- jurisdiction and procedural stage;
- route type;
- Milestone or Return type;
- Incident class.

A pattern in one context should not become a universal conclusion.

## 3. Service evidence and Capability evidence are not the same

MGSN primarily observes service supply and engagement performance.

That evidence may reveal a possible weakness in a Skill, Package, checklist or shared Capability Definition. It does not prove the change by itself.

```text
Provider Performance Evidence
≠ User Capability Evidence
≠ Verified MO Capability

One Provider Outcome
≠ Platform Capability Canon
```

## 4. The governed Capability-learning path

Where evidence appears reusable beyond routing, MGSN should use a controlled handoff:

```text
MGSN Engagement Evidence
→ Rights and Confidentiality Review
→ De-identification where required
→ Attributed Case Candidate
→ Capability Evaluation
→ Principle Candidate or Capability Change Proposal
→ Controlled Version Decision
```

MGSN may submit the evidence and operating context. Capability governance decides whether the evidence is generalizable, compatible and sufficient.

## 5. Public stars distort professional quality

Public ratings collapse service complexity, customer conduct, confidentiality, authority constraints, valid refusal, unresolved Disputes and Evidence quality.

```text
Customer Satisfaction
≠ Professional Quality
≠ Authority Compliance
≠ Safe Fulfillment
```

High-volume Providers may dominate visibility while difficult work appears weaker than simple work.

## 6. Internal classifications can be useful

MGSN may use service-specific classifications such as:

- Recommended Primary;
- Qualified Alternative;
- Specialist Route;
- Pilot Route;
- Operator Review Required;
- Restricted for urgent work;
- Evidence Refresh Required;
- Package Under Review;
- Temporarily Suspended.

These classifications are time-bound, evidence-backed and reviewable. They are not universal ranks.

## 7. Learn from performance and recovery

Positive Evidence includes clear Acceptance, accurate scope, reliable Milestones, complete Returns, safe refusal and effective Correction.

Failure may reflect Provider weakness, ambiguous Package, Instruction error, customer delay, official change, funds control or platform workflow.

```text
Incident Occurrence
≠ Identical Trust Consequence
```

Prompt disclosure and cooperative recovery should be distinguished from concealment or blocked handoff.

## 8. Reflection and Case remain contextual

An engagement may produce a Reflection Candidate or Case Candidate. These should preserve:

- source engagement;
- Provider and Package context;
- Capability and Skill references;
- Evidence and Outcome;
- unresolved responsibility;
- rights and privacy status;
- limits on generalization.

```text
Case Candidate
≠ Shared Knowledge
≠ Canonical Principle
```

## 9. Feedback must be role-aware

A customer, operator, professional reviewer and Finance reviewer observe different parts of the engagement.

Free text may provide context, but it should not directly control routing or Capability status.

## 10. Bias, volume and data controls

Learning may be distorted by geography, language, service complexity, selection bias, operator preference and incomplete migration.

Raw counts are not enough. The network should retain sample size, service mix, severity, confidence, unresolved Incidents and known data gaps.

```text
Data Available to Resolve an Engagement
≠ Permission to Train Every Model
≠ Permission to Publish Every Insight
```

## 11. Relationship provenance is not ownership

```text
Introduced By
≠ Owned By
≠ Universally Endorsed By
```

An introducing Workplace may contribute useful Provider Evidence. It cannot suppress evidence from other engagements or create permanent routing priority.

## 12. Learning should improve controls before labels

Useful outcomes include:

- clearer Service Package templates;
- stronger Evidence requirements;
- better Milestone definitions;
- earlier Authority checks;
- refined Eligibility rules;
- broader Provider recruitment;
- improved replacement packets;
- updated training and Simulation;
- a Capability Change Proposal where evidence justifies one.

The strongest lesson often changes the Product or Workflow rather than a Provider label.

## 13. Contestability remains necessary

Material changes to Eligibility, portfolio role, Trust or learning evidence should be reviewable.

Providers should be able to correct factual errors, provide Evidence and demonstrate remediation.

## 14. Product principle

```text
Bounded Engagement Evidence
+ Service-specific Trust
+ Incident and Dispute Findings
+ Rights, Privacy and Due Process
+ Product-control Improvement
+ Governed Capability-learning Handoff
= Responsible Network Learning
```

MGSN should become more intelligent without becoming a reputation marketplace or a silent author of Capability canon.
