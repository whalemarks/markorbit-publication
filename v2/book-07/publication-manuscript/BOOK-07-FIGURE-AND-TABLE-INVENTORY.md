## 4. Visual design rules

Figures should:

- use the same defined terms as the manuscript;
- show authority and ownership boundaries explicitly;
- avoid implying deployed software screens;
- avoid jurisdiction-specific legal claims;
- distinguish decision, acceptance, instruction, action, Evidence and formal-state update;
- use one consistent notation across the book.

Recommended notation:

```text
solid arrow = governed progression
broken arrow = conditional or review-dependent progression
boundary box = ownership or authority boundary
warning marker = unsafe shortcut
return arrow = Evidence or recovery path
```

## 5. Production priority

Minimum publication set:

```text
Figures 1, 2, 4, 5, 8, 10, 11 and 12
Tables 1, 2, 3, 4, 6, 7 and 8
```

Other visuals may be added after copyedit, but visual work should not reopen substantive architecture without a documented editorial reason.
