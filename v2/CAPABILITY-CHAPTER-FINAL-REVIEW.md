## 4. Final Capability hierarchy

```text
Capability Domain
→ Capability
→ Skill
→ Action / Invocation
```

- Capability Domain groups related Outcomes.
- Capability defines a stable, governed and versioned Outcome contract.
- Skill is a reusable implementation or method.
- Action / Invocation is a bounded use in a specific context.

```text
Capability
≠ Skill
≠ Tool
≠ Workflow
≠ one successful output
```
