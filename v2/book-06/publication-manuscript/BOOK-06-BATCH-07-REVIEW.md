# Book 06 v2 Publication Reconstruction — Batch 07 Review

## Scope reviewed

- CH25 — A Capability Map Must Be Evidence-backed and Perishable
- CH26 — Learning in Work Without Confusing Guidance and Authority
- CH27 — Simulation, Assessment and Certification Boundaries
- CH28 — Paid Tasks Require Eligibility, Scope, Review and Fair Compensation

## Structural verdict

Batch 07 establishes Lite's Capability Center and paid professional Contribution model.

The chapters connect development, assessment and real production without allowing the following categories to collapse:

```text
Learning
≠ Capability
≠ Certification
≠ Eligibility
≠ Assignment
≠ Professional Qualification
≠ Production Authorization
```

The controlled progression is:

```text
Knowledge and practice
→ evidence
→ scoped Capability
→ current eligibility
→ explicit Assignment
→ bounded Contribution
→ review and Outcome
→ correction, compensation and capability update
```

## CH25 review — Capability Map

The chapter correctly replaces profile skill tags with scoped, evidence-backed and perishable Capability records.

It defines a progression from awareness through learning, assisted and supervised work, independent performance, advanced exception handling, review and expert contribution.

Important controls include:

- evidence-type separation;
- jurisdiction, task, language and complexity scope;
- current, refresh-due, restricted, expired, superseded and revoked states;
- error and correction evidence;
- multi-dimensional rather than universal scores;
- private, Workplace and network projections;
- appeal and reassessment;
- separation of Capability from Assignment and authority.

No public universal professional leaderboard is authorized.

## CH26 review — learning in work

The chapter establishes learning at the moment of real difficulty while retaining task-risk, supervision and professional-authority boundaries.

Guidance modes are separated into:

- reference;
- guided preparation;
- simulation;
- supervised production;
- independent work;
- escalation.

The chapter correctly preserves:

```text
Guidance
≠ authorization

Useful learning feedback
≠ accepted production output
```

It also prevents urgent customer obligations from becoming unsafe teaching exercises and requires source-linked, version-aware explanations.

## CH27 review — simulation, assessment and certification

The chapter separates safe practice, defined evaluation and bounded recognition.

Simulation cannot change live customer, payment or formal state. Assessment states what it actually tested and under which assistance conditions. Certification records issuer, scope, evidence, issue date, expiry and limitations.

The following boundary is explicit:

```text
MarkOrbit or Workplace certification
≠ statutory or professional license
```

Certification remains perishable, appealable and insufficient by itself for paid-task or customer-work eligibility.

## CH28 review — paid tasks

The chapter defines a complete paid Contribution chain:

```text
Need
→ Work Package
→ eligibility
→ offer and acceptance
→ Assignment
→ bounded access
→ Contribution
→ review
→ acceptance, revision or rejection
→ compensation and evidence
```

It requires understandable task terms, data minimization, defined review criteria, scoped revision, compensation transparency, rights and confidentiality, conflict controls, failure recovery and AI-use conditions.

It explicitly rejects unpaid production disguised as training or certification.
