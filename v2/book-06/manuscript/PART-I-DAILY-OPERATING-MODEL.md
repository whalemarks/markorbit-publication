# Part I — Daily Operating Model

## Chapter 01 — Lite begins with the day, not the database

Independent trademark professionals rarely begin work by asking which module they should open. They begin with pressure:

- something must be published;
- a customer needs a reply;
- a trademark should be promoted;
- a buyer request arrived;
- an opportunity is expiring;
- a certificate or filing requires follow-up;
- a capability gap blocks progress.

Lite should therefore begin with `Today`, an operating surface that translates data, tasks, opportunities and business context into a small number of meaningful next actions.

The home experience should answer:

```text
What should I publish today?
What can I sell today?
Which demand should I respond to?
Which customer or inquiry needs follow-up?
Which professional risk requires attention?
What should I learn next?
```

## Chapter 02 — Today is a decision layer

Today is not a decorative dashboard.

It should rank candidate actions using:

- urgency;
- commercial value;
- customer commitment;
- deadline risk;
- probability of conversion;
- current Workplace capacity;
- available capability;
- missing information;
- relationship ownership;
- recent user behavior.

A suggested action remains a candidate until the user accepts, edits, dismisses or delegates it.

## Chapter 03 — Different Workplaces need different mornings

A marketing-led Workplace may see:

- recommended topics;
- under-promoted trademarks;
- content performance;
- new buyer requests.

An asset-led Workplace may see:

- stale listings;
- expiring resale authorizations;
- unmatched buyer demand;
- price-review suggestions.

A service-led Workplace may see:

- maintenance opportunities;
- customer follow-up;
- document gaps;
- MarkReg handoffs.

A capability-led individual may see:

- paid tasks;
- review requests;
- simulation recommendations;
- expiring certification.

Lite should adapt through Resource Profile, Need Profile and pipeline state rather than forcing every user into one fixed homepage.

## Chapter 04 — The daily loop

The product should support a repeatable loop:

```text
observe
→ decide
→ create or act
→ follow up
→ record outcome
→ learn
```

Each completed action should improve at least one of:

- customer memory;
- content performance;
- trademark inventory quality;
- request matching;
- transaction pipeline;
- service opportunity quality;
- capability evidence.

## Chapter 05 — Attention must remain bounded

A daily operating product can easily become a source of noise.

Lite should avoid:

- endless notifications;
- artificial urgency;
- repeated generic recommendations;
- content suggestions unrelated to business objects;
- opportunities without source confidence;
- tasks the Workplace cannot lawfully or professionally perform.

The system should prefer a short prioritized list with explanation over a large activity feed.

## Chapter 06 — Opportunity and obligation are different

Lite must visually distinguish:

```text
customer obligation
internal task
commercial opportunity
network invitation
learning recommendation
system warning
```

A missed filing deadline and an unposted social article cannot share the same urgency language.

## Chapter 07 — The user remains the operator

Lite may suggest, organize and automate bounded steps, but the Workplace remains responsible for commercial decisions and professional commitments.

```text
recommended action
≠ accepted instruction

predicted opportunity
≠ confirmed customer intent

automated draft
≠ approved communication
```

## Chapter 08 — Daily value determines retention

Lite succeeds when a user opens it because the product helps answer real operating questions, not because the user fears losing unused credits.

The strongest retention signals are:

- useful work completed;
- customers followed up;
- trademarks promoted;
- requests matched;
- inquiries advanced;
- opportunities converted;
- capabilities improved.
