# Canonical Terminology Register

This register defines the preferred v2 terms for editorial and implementation-facing consistency. It does not activate candidate Core semantics.

## Editorial rules

1. Capitalize named architecture objects consistently in English manuscripts.
2. Do not use “owner” without specifying the authority type.
3. Do not use “professional” to infer qualification or authorization.
4. Do not describe AI output as a decision unless an authorized actor has accepted it.
5. Do not describe a Provider Return as completed formal state before validation.
6. Keep Product-local terms visibly specialized from shared candidate semantics.
7. Use `Mark Global Service Network (MGSN)` on first formal occurrence; use MGSN thereafter.
