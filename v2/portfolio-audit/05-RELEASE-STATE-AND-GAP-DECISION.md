# Release-State and Gap Decision

## Portfolio state after the v2 rewrite

```text
Books 01–07 v2 strategic manuscripts: complete as working edition
Book 02 semantic reconciliation: complete as candidate audit
Cross-book strategic coherence: accepted
Core semantic activation: not granted
Publication authority: not granted
Implementation authority: limited to controlled pilots and existing active contracts
```

## Why v2 should not replace the current baselines yet

The v2 edition contains a coherent future architecture, but three classes of work remain.

### Contract gaps

The following candidate clusters still require field-level mapping against the frozen Core:

1. Work Package / Assignment / Contribution / Outcome;
2. Engagement and engagement-specific roles;
3. Need and its Product specializations;
4. Capability credential and Production Authorization;
5. controlled L/M/R/P vocabularies.

### Editorial gaps

The working manuscripts intentionally use many short chapters to capture the full concept surface. Before publication they require:

- consolidation;
- removal of repeated lock statements;
- consistent examples;
- figure planning;
- Chinese/English terminology strategy;
- reader-specific framing;
- references and source notes where factual claims are made.

### Implementation gaps

The architecture has not yet proved:

- Lite subscription willingness through trademark commerce;
- reliable private/network inventory freshness;
- buyer-request matching and conversion;
- US maintenance simulation-to-production progression;
- contributor quality and compensation operations;
- MGSN controlled-corridor economics and recovery.

## Release-state decision

The recommended state is:

# `v2 Architecture Candidate — Portfolio Coherence Accepted`

This state means:

- v2 may guide controlled planning and specification;
- v2 may not override active Core or frozen publication semantics;
- candidate terms must be labeled in implementation work;
- product pilots should record evidence that informs later semantic activation;
- no Book should be called final or frozen.

## Promotion gates

### Gate A — Contract readiness

- field-level frozen Core audit completed;
- formal Change Proposal decisions recorded;
- accepted candidate contracts versioned;
- downstream impact and migrations documented.

### Gate B — Pilot evidence

- Lite commerce loop produces measurable recurring use or transactions;
- US maintenance capability loop demonstrates simulation, supervised production, review and Outcome evidence;
- customer relationship and authority protections survive real workflows.

### Gate C — Editorial readiness

- chapter consolidation completed;
- terminology register applied;
- diagrams and examples reviewed;
- duplicated explanations removed;
- publication review completed.

### Gate D — Authority decision

Owner explicitly decides whether v2:

- becomes the next active Portfolio Canon and publication baseline;
- remains an architecture candidate;
- is split into normative specifications and explanatory books.

## Current no-go decisions

Do not yet:

- replace the active Canon;
- deprecate frozen Books 01–07;
- implement new Core root objects without proposals;
- create Book 08;
- launch an open task marketplace;
- make MarkReg the compulsory commercial center;
- make MGSN the contracting or funds intermediary in every corridor;
- publish certification as equivalent to professional qualification.
