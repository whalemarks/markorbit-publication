# Book 03 — Chapter and Thesis Map

## Book thesis

Book 03 argues that professional execution is a governed sequence of state transitions in which capability, authority, evidence, review, approval, external effect and formal truth remain distinct.

## Part I — Execution constitution

| Chapter | Main thesis |
|---|---|
| CH01 | Execution is a governed transition, not task completion. |
| CH02 | Execution coordinates work without absorbing business, relationship or formal-state ownership. |
| CH03 | Context, scope and authority must exist before work can safely begin. |
| CH04 | Guide or Product intent must be converted into a governed Capability Request. |

Constitutional lock:

```text
Conversation ≠ Workflow
Prompt ≠ Capability Request
Execution State ≠ Formal State
```

## Part II — Request, implementation and assignment

| Chapter | Main thesis |
|---|---|
| CH05 | Request validation and task-specific Eligibility are separate gates. |
| CH06 | Capability is abstract; Implementation Binding selects a governed human, AI, deterministic or hybrid route. |
| CH07 | Workflow, Work Package, Step, Assignment and Contribution must remain distinct. |
| CH08 | Assignment grants bounded responsibility and least-necessary access, with expiry and revocation. |

Constitutional lock:

```text
Capability ≠ Implementation
Eligibility ≠ Assignment
Assignment ≠ Permanent Role
```

## Part III — Production, judgment and Apply

| Chapter | Main thesis |
|---|---|
| CH09 | Contributions and Provider Returns require validation and typed Outcome Assembly. |
| CH10 | Review is typed, attributable, version-bound judgment. |
| CH11 | Approval is specific authorization for an exact next transition. |
| CH12 | Decision-ready Preview must precede consequential Apply. |

Constitutional lock:

```text
Contribution ≠ Outcome
Review ≠ Approval
Approval ≠ Apply
```
