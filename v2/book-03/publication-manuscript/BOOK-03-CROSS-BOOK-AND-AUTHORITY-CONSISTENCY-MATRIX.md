# Book 03 — Cross-book and Authority Consistency Matrix

## Purpose

This matrix checks whether Book 03 preserves the architectural boundaries established across the MarkOrbit publication set.

## Product and organizational boundaries

| Concern | Book 03 rule | Cross-book consistency |
|---|---|---|
| Workplace | Provides commercial and relationship context | Consistent with Book 01 and Book 05 |
| Product | Presents domain experience and Capability entry points | Consistent with Books 01, 04 and 05 |
| Execution | Coordinates governed work and evidence | Primary Book 03 ownership |
| Owning Service | Validates and mutates formal business state | Consistent with Core and Book 05 |
| MGSN / Provider network | Supplies eligible external professional routes | Consistent with Books 04 and 05 |
| Knowledge / Brain | Owns source-backed knowledge and freshness | Consistent with Book 06 |

```text
Execution Coordination
≠ Product Ownership
≠ Customer Ownership
≠ Formal-state Ownership
```

## Object boundaries

| Objects | Required distinction |
|---|---|
| Guide / Workflow | Guidance does not create an active execution path |
| Capability Request / Assignment | A valid request does not identify a performer |
| Workflow / Work Package / Step | Goal coordination, bounded outcome and execution stage remain separate |
| Assignment / Contribution | Responsibility grant and returned work remain separate |
| Contribution / Outcome | Production return requires validation and assembly |
| Review / Approval | Judgment and authorization answer different questions |
| Apply / External Effect | Attempt does not prove consequence |
| Return / Official Truth | External evidence requires validation |
| Execution State / Formal State | Orchestration cannot silently mutate authoritative business truth |

## Human–AI authority boundaries

| Concern | Book 03 rule |
|---|---|
| Model capability | Does not create invocation or action authority |
| M0–M5 | Defines production mode, not risk or professional status |
| R0–R4 | Defines control depth, not implementation type |
| AI Review | Remains a candidate where qualified human Review is required |
| AI Approval | Cannot be inferred from prediction or sentiment |
| Tool access | Does not create permission for every tool action |
| Memory | Operational context does not create permanent memory or training rights |

```text
Model Can
≠ Model May
≠ Professional Authority
```

## Professional-network boundaries

| Concern | Required distinction |
|---|---|
| Contributor / Provider | Bounded production role versus externally appointed professional role |
| Recommendation / Selection | Suggested route versus chosen route |
| Selection / Appointment | Commercial choice versus formal engagement |
| Appointment / Acceptance | Instructional authority versus provider commitment |
| Acceptance / Action | Commitment does not prove external action |
| Provider Return / Official Truth | Provider statement requires official or Owning Service validation |

## Customer and relationship boundaries

```text
Customer Data Access
≠ Customer Contact Permission
≠ Customer Relationship Ownership
```

Book 03 preserves the Relationship Owner in white-label, co-delivery and partner-led work. Delivery Owner status does not transfer the customer, contract or marketing relationship.

## Financial boundaries

| Concern | Book 03 rule |
|---|---|
| Customer payment | Does not prove provider or official payment |
| Payment received | Does not establish professional readiness or filing Approval |
| Funds checkpoint | Can be coordinated by Execution |
| Payment custody | Not created by orchestration or integrated UX |
| Contributor compensation | Remains linked to bounded production and contractual terms |
| Revenue share | Does not transfer authority or relationship ownership |

```text
Payment Orchestration
≠ Payment Custody
```

## Data, privacy and learning boundaries

```text
Operational Access
≠ Reuse Right
≠ Shared Knowledge
≠ AI-training Permission
```

Simulation fixtures, public cases, assessment scenarios and training datasets require independent rights and privacy review.
