## Architecture corrections confirmed

### Guide versus Execution

```text
Guide
= understanding, explanation and decision preparation

Execution
= governed production, review, approval, action and evidence
```

A conversational sequence is not automatically a Workflow, and an AI answer is not an Approval.

### Capability Request versus Workflow

```text
Capability Request
= governed request for an outcome

Workflow
= coordination of a goal and its stages
```

One Workflow may invoke several Capabilities. A bounded Capability may execute without a large Workflow.

### Review versus Approval

Review produces findings. Approval authorizes an exact version and next boundary. The records remain distinct even when one qualified actor performs both roles.

### Apply versus protected action

Apply is the governed move from approved preparation toward effect. External Protected Action is a consequential external action class requiring explicit authority, idempotency and evidence. The concepts overlap in some paths but are not synonyms.

### Execution versus formal-state ownership

Execution owns coordination evidence. Workplace retains business sovereignty. Core remains semantic authority. Owning Services retain formal-state mutation responsibility. Official truth remains grounded in authoritative evidence.
