# Book 03 v2 — Publication Closing

## Execution is where intelligence becomes consequence

Book 03 establishes the constitutional execution model for MarkOrbit. Its subject is not task management, automation or AI productivity in isolation. Its subject is the governed transition from intent to consequence.

```text
Understand
→ Request
→ Validate
→ Bind
→ Assign
→ Produce
→ Review
→ Synthesize
→ Preview
→ Approve
→ Apply
→ Observe
→ Reconcile
→ Formalize
→ Learn
```

Each transition preserves its own actor, authority, evidence, version, risk and recovery path.

## The central distinction

```text
Capability
≠ Authority
```

A human, AI model, deterministic program or provider may be able to perform a task without being authorized to make the decision, communicate externally, move funds, represent a customer, submit to an authority or mutate formal state.

Book 03 therefore keeps the following boundaries permanent:

```text
Guide ≠ Workflow
Prompt ≠ Capability Request
Capability Request ≠ Assignment
Assignment ≠ Contribution
Contribution ≠ Outcome
Review ≠ Approval
Approval ≠ Apply
Apply ≠ External Effect
Provider Return ≠ Official Truth
Execution State ≠ Formal State
```

## Human–AI work is governed by mode and risk

M0–M5 describe how work is produced. R0–R4 describe what control is required before reliance or action.

```text
M-mode ≠ R-tier
```

A highly automated operation can require R4 control. A human-only operation can be low risk. The system therefore classifies the proposed transition rather than labeling an entire Product, profession or model as safe.

## Professional authority remains external to platform confidence

Certification, Assessment Evidence, Production Authorization, user status, platform reputation and model performance cannot create a legal or professional qualification.

```text
Assessment Passed
≠ Professional Qualification
≠ Professional Authority
```

Where local professional authority is required, MarkOrbit must identify the qualified person or provider, current credential, appointment, conflict status and exact action authorized.

## External action requires a visible boundary

The decision-ready Preview links the reviewed object, approved object and applied payload.

```text
Previewed Payload
= Approved Payload
= Applied Payload
```

Material change requires re-review or re-approval. Every consequential Apply requires authority checks, idempotency, an Evidence Contract, explicit Unknown handling and a recovery route.

## Unknown is a valid state

A timeout, silent provider, missing receipt or conflicting external record does not establish success or failure.

```text
Unknown
≠ Failure
≠ Success
≠ Permission to Retry
```

The correct path is preservation of evidence, reconciliation and a new authorized decision.

## Execution coordinates but does not absorb ownership

Execution can coordinate Work Packages, Assignments, Contributions, Reviews, Approvals, external handoffs, Returns and recovery. It does not automatically own the Customer, Order, Matter, Payment, official right, relationship or professional engagement.

```text
Orchestration
≠ Relationship Ownership
≠ Professional Authority
≠ Formal-state Authority
```

## Simulation and production are separate worlds

Simulation can reproduce the complete structure of governed work while blocking live customers, real providers, official submissions, production credentials, real payments and production-state mutation.

```text
Simulation Approval
≠ Production Approval
```

Capability Evidence may support Certification and later Production Authorization, but each transition remains explicit and reversible.

## Observability protects both people and institutions

A trustworthy audit trail must reconstruct what was requested, known, inferred, produced, reviewed, approved, applied and externally observed. It must also preserve manual actions, AI provenance, provider evidence, financial evidence and formal-state validation.

```text
Activity Log
≠ Execution Observability
```

## Learning improves policy without rewriting history

Validated outcomes may produce contextual learning signals. Those signals can become reviewed candidates for new policies, templates, routing rules, scenarios or Knowledge Objects.

```text
Historical Event
≠ Learning Signal
≠ Policy
≠ Canonical Truth
```

New policy versions do not silently reinterpret historical execution.

## The constitutional promise

Book 03 does not promise that MarkOrbit will eliminate uncertainty, professional disagreement, provider failure or human error. It defines how the system behaves when those realities occur.

The final execution equation is:

```text
Trusted Execution
= Capability
+ Context
+ Authority
+ Evidence
+ Review
+ Recovery
+ Observability
```

The system is trustworthy when it can move quickly without hiding who decided, who was authorized, what changed, what evidence exists and what must happen when reality does not match the expected path.
