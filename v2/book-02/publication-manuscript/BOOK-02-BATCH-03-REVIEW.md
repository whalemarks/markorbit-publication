## Main decisions

### 1. Product composition does not collapse architectural layers

```text
Product
≠ Capability
≠ Workflow
≠ Tool
≠ Service
≠ Implementation
```

Capability remains the durable outcome contract. Product remains the user and commercial arrangement. Workflow remains orchestration. Tool remains an execution instrument. Service meanings must identify runtime, Product service, Owning Service or external professional service explicitly.

### 2. Work and business records remain separate

```text
Order
≠ Matter
≠ Work Package
≠ Assignment
```

Order preserves the commercial promise. Matter preserves the long-lived professional subject. Work Package defines bounded production. Assignment grants scoped authority to perform it.

Work Package and Assignment remain candidate shared semantics and were not activated.

### 3. Trust records preserve the path to formal state

```text
Evidence
≠ Claim
≠ Decision
≠ Formal State
```

Evidence supports a proposition. Claims state what is asserted and with what epistemic status. Decisions are attributable judgments. Formal State belongs to the applicable Owning Service.

```text
Provider Return
≠ accepted truth

Decision recorded
≠ external action performed

Accepted Outcome
≠ formal-state mutation
```

### 4. Controlled movement does not transfer ownership

```text
Projection
≠ Handoff
≠ Return
≠ Transfer
≠ Formal-state Mutation
```

Projection controls visibility. Handoff transfers bounded work context. Return carries evidence or result to the origin. Correction repairs accepted meaning while preserving history.

The originating context validates Return before formal acceptance.
