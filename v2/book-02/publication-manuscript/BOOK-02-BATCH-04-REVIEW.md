## Main decisions

### 1. Capability remains the stable contract

```text
Capability Definition
= durable governed outcome contract
```

Product, Workflow, Skill, Tool and Implementation may change without silently redefining the Capability.

### 2. Implementation concepts remain separate

```text
Skill
≠ Tool
≠ Service
≠ Implementation Profile
≠ Capability
```

An Implementation Profile binds a Capability version to a governed realization, but does not create Eligibility, Assignment or authority.

### 3. Proficiency, collaboration and control use separate vocabularies

```text
L-level
≠ M-mode
≠ R-tier
```

L-level describes demonstrated Capability. M-mode describes human–AI composition. R-tier describes required control.

L1–L5, M0–M5 and R0–R4 remain candidate controlled vocabularies subject to frozen mapping and formal activation.

### 4. Credentials and permissions remain separated

```text
Capability Evidence
≠ Certification
≠ Professional Qualification
≠ Production Authorization
≠ Eligibility
≠ Assignment
≠ Professional Authority
```

MarkOrbit assessment cannot create legal qualification. Production Authorization permits bounded real work but does not authorize a specific task or protected action.

## Cross-book consistency

### Book 03 — Execution

Book 03 continues to own runtime validation, Implementation Binding, Eligibility, Assignment, M-mode application, R-tier control, Review, Approval and Apply.

Book 02 only defines the stable semantic distinctions.

### Book 04 — Capability and professional supply

Book 04 may use Capability Definitions, assessment evidence, Certification and authorization records to organize professional supply. It cannot convert marketplace status or performance score into Professional Authority.

### Book 05 — MarkReg

MarkReg may package Capabilities into lifecycle services and route them through human, AI and Provider implementations. Product promises cannot broaden Capability authority or bypass professional review.

### Book 06 — Lite

Lite may expose Capability experiences, recommendations and opportunities. Access to a Product capability does not become Production Authorization, Assignment or external-action authority.
