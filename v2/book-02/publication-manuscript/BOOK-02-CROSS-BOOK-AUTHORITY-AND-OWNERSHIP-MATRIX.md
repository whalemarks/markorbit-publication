# Book 02 — Cross-book Authority and Ownership Matrix

## Purpose

This matrix prevents Book 02 shared semantics from absorbing the Product, execution, relationship, knowledge or formal-state responsibilities defined elsewhere.

| Concern | Book 02 / Core | Book 03 / Execution | Book 05 / MarkReg | Book 06 / Lite | Data / Knowledge / Brain | Owning Service / external authority |
|---|---|---|---|---|---|---|
| Shared meaning | defines identifiers, contracts, lifecycles and authority distinctions | consumes contracts | consumes and profiles contracts | consumes and profiles contracts | consumes and publishes typed references | does not redefine Core meaning |
| Customer relationship | defines Customer, Contact and Relationship references | preserves relationship constraints during work | may deliver lifecycle services | may identify and nurture opportunities | may hold purpose-limited context | originating Workplace remains Relationship Owner unless explicitly changed |
| Commercial commitment | defines Order and Settlement references | coordinates authorized work | owns Product-specific offers and service delivery | owns Lite-specific offers and marketplace experience | does not create commercial commitments | contracting parties and payment services establish formal commercial facts |
| Professional Matter | defines shared Matter reference and boundaries | coordinates Work Packages and transitions | composes trademark lifecycle experience | may hand opportunities into MarkReg | provides evidence and intelligence | Matter Owning Service validates formal state |
| Work execution | defines shared semantic distinctions and candidate contracts | owns validation, binding, eligibility, Assignment, Review, Approval, Apply and reconciliation | supplies Product workflows and requirements | supplies content and marketplace workflows | supplies Data, Knowledge and Brain inputs | external provider or official system may perform protected action |
| Data | defines reference and provenance contracts | uses minimum necessary data | consumes domain data | consumes public and commercial data | Data service owns source capture, transformation and release | official source remains authoritative for official facts |
| Knowledge | defines interoperable Source, Claim, Version and Citation references where admitted | attaches Knowledge to work and review | uses jurisdictional and provider knowledge | uses source-backed knowledge for content and opportunities | Knowledge service owns document, Claim and version lifecycle | qualified professional owns reserved interpretation where applicable |
| Brain | defines typed result profiles and non-canonical boundary | governs use of results in production | uses recommendations and candidates | uses retrieval and generation | Brain runtime produces typed results | cannot create customer instruction, approval or formal state |
| Provider network | defines Organization, role, qualification and appointment references | validates eligibility and governs Handoff / Return | selects routes for trademark delivery | may expose supply or opportunity views | provides supporting data and intelligence | Provider Organization and responsible professional bear external responsibility |
| Payment | defines neutral references and state distinctions | coordinates approved transitions and evidence | owns Product pricing and billing policy | owns Lite commercial policy | may support fee knowledge and reconciliation data | payment processor, contracting party and financial system establish settlement facts |
| Formal state | defines state and authority contracts | never acquires formal-state ownership by coordination | displays and acts through Matter services | displays projections | supplies evidence and candidates | Owning Service validates and mutates; official institution establishes external fact |

## Permanent cross-book locks

```text
Core Contract ≠ Execution Runtime
Core Object ≠ MarkReg Product Record
Core Reference ≠ Knowledge Repository
Lite Opportunity ≠ Customer Instruction
Data Product ≠ Official Authority
Brain Result ≠ Formal State
Provider Return ≠ Accepted Truth
Payment Orchestration ≠ Payment Custody
```

## Book 03 boundary

Book 02 may define the shared meanings of Capability, Work Package, Assignment, Contribution, Review, Outcome, Handoff and Return. Book 03 owns the governed runtime sequence.

```text
Book 02 defines what the records mean.
Book 03 governs how consequential transitions occur.
```

No Book 02 chapter authorizes unsupervised Apply, protected external action, retry or formal-state mutation.

## Book 05 boundary

MarkReg owns the international trademark lifecycle experience, including filing preparation, prosecution, maintenance, transactions, customer communication and provider coordination.

Book 02 does not absorb:

- filing-package policy;
- jurisdiction-specific checklists;
- customer conversion rules;
- provider procurement packages;
- deadline service policy;
- MarkReg pricing or billing.

These may reference Core contracts without becoming Core semantics.

## Book 06 boundary

Lite owns daily growth, content, trademark supply and demand, marketplace and capability experience.

Book 02 does not turn Listings, Buyer Inquiries, Transaction Opportunities, Content Briefs or Campaign Assets into shared root objects merely because they use Customer, Trademark, Opportunity, Projection or Evidence references.

## Knowledge and Brain boundary

Knowledge organizes sourced Claims. Brain produces context-specific typed intelligence. Core provides stable interoperability references where necessary.

```text
Knowledge Accepted ≠ Business State Changed
Brain Recommendation ≠ Customer Decision
Operational Access ≠ Training Permission
```

## Authority conclusion

```text
Shared visibility does not transfer ownership.
Shared semantics do not create shared write authority.
Coordination does not create formal-state authority.
Capability does not create professional authority.
```
