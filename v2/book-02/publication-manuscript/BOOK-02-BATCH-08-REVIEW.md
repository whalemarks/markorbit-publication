## Main decisions

### 1. Projection is governed visibility

```text
Projection
≠ Source Object
≠ Ownership Transfer
≠ Relationship Transfer
≠ Mutation Authority
≠ Unlimited Reuse
```

Projection must preserve source, version, purpose, visible fields, permitted actions, expiry, freshness, rights and correction behavior.

Demand, Capability, Asset, Audience and Capacity forms remain profiles over one Projection mechanism unless stronger interoperability needs emerge.

### 2. Public availability does not eliminate governance

```text
Publicly Accessible
≠ Current
≠ Verified for Every Use
≠ Marketing Consent
≠ Unrestricted Profiling Permission
```

Public trademark and provider data may support research or opportunity discovery without creating a Customer, Relationship or permission to contact.

### 3. Handoff and Return are separate boundary contracts

```text
Handoff
≠ Message Forward
≠ Assignment
≠ Provider Appointment
≠ External Action

Return
≠ Reply Message
≠ Accepted Outcome
≠ Official Truth
```

Handoff carries bounded context and responsibility outward. Return carries typed result and evidence inward.

Destination acknowledgement, acceptance, action and official effect remain separate states.

### 4. Return validation remains at the origin or Owning Service

```text
Return Received
≠ Return Validated
≠ Outcome Accepted
≠ Formal State Updated
```

Evidence Return contracts should be defined before consequential work. Unknown Return remains distinct from success, failure and safe retry.

### 5. Contributor and Provider remain distinct

```text
Contributor
≠ Provider
≠ Professional Authority
≠ Employee
≠ Reviewer
≠ Delivery Owner
```

Contributor supplies bounded work under Assignment or Engagement context. Provider is an Organization profile carrying an external service responsibility under selection, appointment and acceptance rules.

```text
Provider Organization Eligible
≠ Every Employee Qualified

Qualified Professional
≠ Appointed for This Matter
```

### 6. Appointment and Assignment remain separate

```text
Provider Appointment
≠ Assignment
```

Provider Appointment establishes an external service route. Assignment grants responsibility for an exact Work Package. One may reference the other without sharing its lifecycle or authority.

### 7. Public, shared and local records require explicit classification

```text
Public Contract or Projection
Shared Semantic Record
Product-local Record or Profile
```

Public records are governed disclosures. Shared records carry stable interoperable meaning. Product-local records support differentiated workflows, interfaces and policy.

```text
Widely Used
≠ Core-owned

Product-local
≠ Semantically Ungoverned
```

### 8. Product-local profiles should not multiply Core roots

Records such as Listing, Buyer Inquiry, Transaction Opportunity, Candidate Route Set, Provider Package, Case Fixture and Commission Claim remain Product-local or profile concepts unless they pass the F0–F4 admission process.

```text
New Noun
≠ New Root Object Required
```

### 9. Shared semantics do not create shared write authority

```text
Shared Semantics
≠ Shared Write Authority
```

Every shared formal record still requires an Owning Service, lifecycle and correction route.

### 10. Migration from local to shared must be governed

```text
Local Schema Inventory
→ Semantic Comparison
→ Authority and Lifecycle Design
→ Change Proposal
→ Conformance Fixtures
→ Mapping and Backfill Plan
→ Compatibility Window
→ Consumer Migration
→ Deprecation Evidence
```

Publication prose and repeated Product use do not perform this migration.

## Cross-book consistency

### Book 03 — Execution

Book 03 continues to own:

- Handoff acceptance runtime;
- Assignment;
- external protected actions;
- Return validation, retry and reconciliation;
- provider silence and replacement recovery.

```text
Core Boundary Contract
≠ Execution Runtime
```

### Book 05 — MarkReg

MarkReg may use customer, trademark, Matter, filing, provider and lifecycle Projections. Product Handoff does not transfer the customer relationship or formal Matter authority.

### Book 06 — Lite

Lite may publish public content, trademark Listings, service signals and Opportunity profiles. Public data does not create contact consent, and Product-local commerce records do not become new Core roots.

### Workplace and provider network

White-label and co-delivery arrangements must preserve Relationship Owner, Delivery Owner, Provider, Professional Authority, Contracting Party and Payment Receiver as distinct contextual roles.
