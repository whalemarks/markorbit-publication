## 3. CP-A finding

The frozen baseline already supplies a strong authority stack:

```text
Identity recognizes.
Capability describes possible action.
Permission authorizes a controlled action.
Policy evaluates context.
Human Review records review.
Task carries exact-work assignment references.
Owning Services execute and mutate state.
```

It does not fully represent the externally grounded professional-authority chain:

```text
qualification
+ jurisdiction
+ verification freshness
+ restrictions
+ conflict clearance
+ appointment
+ Matter scope
+ authority to advise, sign, submit or represent
```

### CP-A dispositions

| Candidate | Disposition |
|---|---|
| Capability Evidence | F1/F2 Evidence profile; no proposal |
| Capability Certification | F2/F3 profile study; no root-object proposal yet |
| Professional Qualification | narrow external-reference pre-proposal research |

| task-specific Eligibility | Book 03 runtime decision profile |
| Assignment | deferred to CP-B |
| Professional Authority | narrow F3/F4 pre-proposal research with legal review |

## 4. CP-A rejected collapses

The research rejected:

```text
Certification = Permission
Qualification = Identity verification level
Human Review completion = Professional Authority
Task assignment = general Production Authorization
one universal Authority Object
```

The preserved chain is:

```text
Identity
→ Capability and Evidence
→ Certification profile
→ external Qualification reference
→ Production Authorization permission profile
→ runtime Eligibility
→ exact Assignment
→ separately grounded Professional Authority
```

## 5. CP-A field-level result

Many Production Authorization fields already map to frozen Permission:

- subject;
- action;
- resource;
- Product, Workflow, Service, API and DataAccess scope;
- effect and status;
- organization context;
- Capability and Policy references;
- source and audit information.

Potential gaps include:

- effective interval;
- model, rule or service version binding;
- supervision requirement;
- M-mode and R-tier;
- independent suspension consequences for active Assignments.

This supports testing Production Authorization as a Permission profile before proposing a new Object or changing frozen Permission semantics.

## 6. CP-B finding

The frozen production model already contains:

```text
Matter
→ Workflow Contract
→ Task
→ Document / Evidence / Communication
→ Human Review
→ Owning Service result
→ Event trace
```

Task already contains scope-related references, assignee, responsibility, dependencies, deadline, workflow and artifact links.

The v2 chain remains useful where it preserves distinct production boundaries:

```text
Work Package
→ Assignment
→ Contribution
→ Review
→ Outcome
```

but the research did not prove that every term requires a new root Object.

### CP-B dispositions

| Candidate | Disposition |
|---|---|
| Work Package | F2 Task/Workflow profile research |
| simple internal Assignment | retain Task assignee/responsibility fields |
| bilateral or cross-Workplace Assignment | F3 pre-proposal research after consumer proof |
| Contribution | F2 submission profile over Document/Evidence/Event |
| Review | retain frozen Human Review Contract |
| Outcome | F2 acceptance profile; F3 only after consumer proof |

## 7. CP-B rejected collapses

The research rejected:

```text
rename Task to Work Package
Task Completed = Outcome
Document = Contribution in every case
new generic Review Object
Workflow Contract owns every production state
trademark Assignment Workflow = generic work Assignment
```

Required state separation remains:

```text
Assignment State
≠ Task State
≠ Work Package State
```

```text
Contribution State
≠ Review State
≠ Outcome State
```

```text
Outcome Accepted
≠ Task Completed
≠ Matter Resolved
≠ Formal State Updated
```

## 9. Why no formal proposal opens yet

Composition and profile alternatives remain viable. The evidence package is not yet complete for:

- legal and professional authority analysis;
- source and privacy handling of credentials and disciplinary status;
- independent consumer proof;
- exact compatibility and migration impact;
- executable positive and negative fixtures;
- Book 03 runtime integration proof.

An omnibus proposal would incorrectly combine external credentials, platform Certification, Permission, Execution Eligibility, Task Assignment, professional authority and production outcomes.
