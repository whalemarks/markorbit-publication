This candidate freezes the current publication reconstruction as a stable editorial and research reference while preserving `B02-BASELINE-V0.1` as the normative Core authority.

## 2. Freeze scope

The candidate scope includes:

- the 36 long-form publication chapters;
- reconstruction charter and source integration records;
- batch reviews and closing package;
- frozen-baseline inventory and disposition matrices;
- candidate-to-frozen mapping;
- current Core implementation mapping;
- authority drift and Change Proposal priority records;
- CP-A and CP-B pre-proposal research;
- noncanonical research fixtures;
- downstream task specifications;
- evidence intake, evaluation, semantic-delta and pause/resume controls;
- Batch 16 status, handoff and dependency records.

## 3. Freeze meaning

Publication freeze means:

```text
The first publication reconstruction is editorially closed.
Its boundaries and candidate dispositions are stable.
Further semantic claims require new evidence and a new controlled workstream.
```

It does not mean:

```text
v2 replaces B02-BASELINE-V0.1
candidates become active Core
research fixtures become contracts
markorbit-core must migrate
Book 03 runtime is approved
production readiness is accepted
```

## 4. Allowed post-freeze maintenance

Allowed without reopening substantive reconstruction:

- spelling, punctuation and formatting corrections;
- broken-link repair;
- navigational indexes;
- metadata and status clarification;
- exact citation insertion that does not change meaning;
- correction of objectively wrong repository identifiers;
- received-evidence references;
- explicit supersession notices;
- rendered publication fixes.

Every maintenance change must state why it is meaning-preserving.

## 5. Changes that reopen the publication freeze

A new controlled editorial workstream is required when a change:

- adds a substantive chapter;
- changes a constitutional boundary;
- changes candidate disposition;
- states that a candidate is active;
- changes an ownership or authority conclusion;
- revises the frozen-baseline mapping;
- changes F0–F4 classification;
- incorporates accepted downstream evidence into a new semantic conclusion;
- proposes replacement of the frozen Book 02 corpus.

## 6. Evidence-triggered addenda

Downstream research should be received as addenda rather than silently rewriting the frozen reconstruction.

Recommended pattern:

```text
Evidence Return
→ Intake Record
→ Evaluation Rubric
→ Semantic Delta Decision Log
→ Addendum or Pre-proposal
→ separate owner decision
```

The original publication conclusion remains historically visible.
