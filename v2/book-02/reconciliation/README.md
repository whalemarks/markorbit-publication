# Book 02 v2 Semantic Reconciliation

## Status

- Edition: v2 working reconciliation
- Authority: draft only
- Frozen baseline: unchanged
- Active Canon: unchanged
- Purpose: determine which v2 concepts are already represented by the frozen Core, which can be expressed by composition, which need explanatory profiles, and which may require a formal Core Change Proposal

## Governing rule

Book 02 is not being freely rewritten.

The accepted Core remains the semantic authority until a separate, explicit change process modifies it.

This package therefore uses five classifications:

```text
F0 — already covered by frozen Core
F1 — covered through composition of existing Core concepts
F2 — explanatory profile or controlled specialization; no Core semantic change
F3 — candidate shared semantic addition
F4 — formal Core Change Proposal required before activation
```

## Non-goals

This package does not:

- mutate frozen Book 02 text;
- promote v2 object names into active Core;
- require every Product object to become a Core object;
- assign formal-state ownership to Core;
- make Brain output canonical;
- make certification equivalent to professional qualification;
- create implementation code;
- decide migration dates.

## Decision gate

After Owner review, each F3 or F4 candidate must be handled separately. No candidate becomes active merely because this reconciliation package is merged.
