# Ideas

## Purpose

This area stores publication idea intake material before it becomes accepted MarkOrbit architecture, book content, research work, or implementation work.

Idea intake keeps early concepts visible without allowing them to silently expand Book 02 Core, Book 02 MVP, Book 03 execution scope, or product implementation scope.

## Status Types

| Status | Meaning |
|---|---|
| Idea | A proposed concept, capability, boundary, or question that needs classification. |
| Accepted architecture | A reviewed and approved architecture decision owned by the appropriate book layer. |
| Book content | Manuscript or planned manuscript content approved for a specific book. |
| Research item | A pre-research or evaluation topic that may inform future decisions but is not canonical architecture. |
| Implementation task | Product or code work approved through a separate implementation process. |

## Intake Review Flow

Idea
→ Intake
→ Classification
→ Publication decision
→ Book planning update
→ Review
→ Rewrite if approved

Intake records are planning artifacts. They do not authorize manuscript rewrites, Core spec changes, MVP expansion, product implementation, or technical POCs by themselves.
